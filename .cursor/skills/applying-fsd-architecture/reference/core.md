# Справка по FSD

## Слои (Layers)
FSD определяет 7 стандартизированных слоёв (сверху вниз):

- app — точка входа в приложение, маршрутизация, провайдеры
- pages — экраны по маршрутам
- widgets — независимые крупные UI-блоки, лейауты приложения 
- features — единицы функциональности, предоставляющие ценность пользователю
- entities — бизнес-сущности (User, Product и т.д.)
- shared — переиспользуемые утилиты и UI-компоненты

## Слои (сверху -> вниз)

```
app → pages → widgets → features → entities → shared
         ↓
   Верхний слой может импортировать только нижележащие слои
   Импорт между слайсами внутри одного слоя запрещён. Однако, если это необходимо для сокращения кода или упрощения логики и прозрачности зависимостей импорт между слайсами внутри одного слоя допустим только с одобрения пользователя. Для этого он должен ответить "Действуй", "Согласен" или что-то в этом роде
```

Правила зависимостей:
- Верхний слой может импортировать только из нижних слоев
- Нижний слой не должен импортировать из верхних слоев
- Импорты между слайсами одного слоя запрещены. Однако, если это необходимо для сокращения кода или упрощения логики и прозрачности зависимостей импорт между слайсами внутри одного слоя допустим только с одобрения пользователя. Для этого он должен ответить "Действуй", "Согласен" или что-то в этом роде
  
## Типичные сегменты

- `ui` - UI-компоненты, стили, визуальная композиция
- `model` - управление состоянием, бизнес-логика, сценарии использования
- `api` - внешние запросы/контракты
- `lib` - локальные утилиты
- `config` - константы и конфигурация

## Быстрый гайд по размещению

- Если переиспользуемо и не зависит от бизнес-домена -> `shared`
- Если специфично для домена и переиспользуется между фичами -> `entities`
- Если это пользовательская возможность/действие -> `features`
- Если это крупный блок, собранный из `features`/`entities` -> `widgets`
- Если это экран маршрута -> `pages`
- Если это инициализация приложения/провайдеры/роутинг -> `app`

## Чеклист перед слиянием

- [ ] Нет нарушений направления зависимостей
- [ ] Нет кросс-импортов между слайсами одного слоя
- [ ] Ответственности сегментов не смешаны
- [ ] Публичный API/пути импортов соответствуют стилю репозитория
- [ ] Линт/типовые проверки проходят
