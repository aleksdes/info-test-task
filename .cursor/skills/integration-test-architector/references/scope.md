# Интеграционные тесты vs unit в этом проекте

## Unit (`__unit-tests__`)

- Изолированный модуль: функция, хук, маппер, редьюсер.
- Минимум окружения; моки узкие и локальные.

## Integration (`__integration-tests__`)

- **Несколько частей слайса** (или явная сборка сегментов) в одном сценарии: например форма + модель + мок `api/`.
- Часто используются Testing Library, роутер, провайдеры (стек из `package.json`) — всё ещё **без реальной сети** к API.

## Пересечение

- Один и тот же баг можно поймать и unit, и integration. Дублировать не обязательно: integration — для **сквозного поведения**, unit — для **деталей логики**.
- Оба слоя: моки API, потолок покрытия **не выше ~90%** — см. [coverage.md](coverage.md).
