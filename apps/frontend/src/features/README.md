Слой фич, функций. Фича это конкретное действие пользователя, действие которое изменяет
состояние.

Каждая фича называется срезом слоя, срез слоя фактически из себя представляет директорию,
названную по имени фичи.

| Cross импорт между срезами              | Импорт других слоев                                            | Структура                              |
|-----------------------------------------|----------------------------------------------------------------|----------------------------------------|
| <span style="color:red">Запрещен</span> | <span style="color:yellow">Только `shared` и `entities`</span> | <span style="color:red">Строгая</span> |

# Файловая структура среза
- `model/` - Директория с кодом который реализует работу с сущностями
- `lib/` - Опционально. Код поддерживающий работы кода из `model`, в фичах это этот сегмент экзотический
  и зачастую он не нужен
- `ui/` - Vue компоненты, могут представить фичу и как то отреагировать на его действие 
  (например кнопки удаления, формы, диалоговые окна)
- `index.ts` - Реэкспорт публичного API


# Ответственность
В зоне ответственности слоя реализация действия пользователя и ответ на его действие.
Фича использует и вызывает методы из слоя `entities/`. 
В то время так как `entities/` только предоставляет методы манипулации, фичи их вызывают в ответ
на действия пользователя, например удаление, отправка формы.

У фичи есть точка входа, то есть фича должна полноценно реализовывать действие, на примере 
фичи удаления пользователя с диалогом подтверждением, точкой входа будет кнопка удаления, которая
вызывает диалоговое окно, после подтверждения в окне, фича вызывает метод удаления у сущности,
отвечает пользователю визуализацией ожидания выполнения действия (блокировка формы, 
сообщение о выполнении или отображение анимации лоадера), так фича становится полноценной и 
проводит пользователя через весь путь выполнения действия.
