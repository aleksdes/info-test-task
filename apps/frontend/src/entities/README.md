Слой сущностей, к сущностям относятся единицы данных аналогичных тем что есть в зоне бэкенда, например
`User, Session, Settings`

Каждая сущность называется срезом слоя, срез слоя фактически из себя представляет директорию,
названную по имени сущности.


| Cross импорт между срезами              | Импорт других слоев                               | Структура                              |
|-----------------------------------------|---------------------------------------------------|----------------------------------------|
| <span style="color:red">Запрещен</span> | <span style="color:yellow">Только `shared`</span> | <span style="color:red">Строгая</span> |

# Файловая структура среза
- `model/` - Директория с кодом который реализует стор сущности, методы ее мутации и геттеры
- `lib/` - Опционально. Код поддерживающий работы кода из `model`, например если есть
  нестандартная реализация загрузки сущности, валидаторы данных, нормализаторы
- `ui/` - Vue компоненты, которые могут представить сущность визуально, но без логики взаимодействия,
  их можно рассматривать как шаблоны для отображения сущностей
- `index.ts` - Реэкспорт публичного API


# Ответственность
В зоне ответственности слоя хранение и чтение или загрузка сущностей с сервера, 
предоставление методов мутации низкоуровневых методов мутации на сервер, 
в отличие от слоя `features/` в сущностях н может существовать код реализующий 
действия пользователя приложения, методы мутации сущностей вызываются из слоя `features/`

На примере сущности `User`, это будет стор реализующий хранение данных пользователя и методы
CRUD операций над сущностью
